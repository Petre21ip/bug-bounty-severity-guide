Bug Bounty Severity Guide - Full Reference (English)

IDOR (Insecure Direct Object Reference)

⬜ Informative: Public data accessible, no impact. 🟨 Low: Access non-sensitive private data (avatar, descriptions). 🟩 Medium: Access personal data (email, phone). 🟦 High: Modify another user's data via PUT/PATCH. 🟪 Critical: Delete/modify full resources (DELETE).
💡 Exploit Tips: Check object IDs in URLs, JSON bodies, or parameters. Look for predictable numeric IDs or GUIDs. Common endpoints: `/user/{id}`, `/orders/{id}`, `/profile/{id}`. Test modification, access, or deletion of resources belonging to other users.

LFI (Local File Inclusion)

⬜ Informative: Error-based only, confirms vector. 🟨 Low: Non-sensitive files (own logs). 🟩 Medium: Access `/etc/passwd`. 🟦 High: Access config files with API keys / DB credentials. 🟪 Critical: LFI → RCE (log poisoning, PHP wrappers).
💡 Exploit Tips: Check parameters like `file=`, `page=`, `template=`. Try null byte bypass, directory traversal (`../`). Look for error messages revealing paths. Often found in legacy PHP apps or poorly validated include mechanisms.

RCE (Remote Code Execution)

⬜ Informative: Error without execution. 🟨 Low: Limited command execution, unclear output. 🟩 Medium: Commands with output (whoami, id). 🟦 High: Read system files (`/etc/passwd`). 🟪 Critical: Reverse shell / full execution.
💡 Exploit Tips: Look at input fields that interact with OS (file upload, ping, shell commands). Check unsanitized system calls. Webhooks, cron endpoints, and legacy scripts are common targets.

SQL Injection

⬜ Informative: Error messages only, confirm vector. 🟨 Low: Blind SQLi, time-based, no data extracted. 🟩 Medium: Enumerate databases and tables. 🟦 High: Extract sensitive data (usernames, emails). 🟪 Critical: Password hashes, write/delete data.
💡 Exploit Tips: Test parameters like `id=`, `search=`, `filter=`. Use error-based, union-based, and time-based payloads. Tools: sqlmap, manual payloads. Common endpoints: login, search, reports, filters.

XSS (Cross-Site Scripting)

⬜ Informative: Filtered/escaped reflected, no impact. 🟨 Low: Reflected, works only on old browsers or own input. 🟩 Medium: Reflected valid → alert(1) works. 🟦 High: Stored, affects other users. 🟪 Critical: Stored + exfiltrate tokens / sessions.
💡 Exploit Tips: Test input fields (forms, search, comments). Check reflected responses, cookies, headers. Try event handlers, script tags, DOM injection. Stored XSS often in comment sections, profiles, message boards.

CORS Misconfigurations

⬜ Informative / N/A: Vector confirmed, no real impact (headers only). 🟨 Low: Non-sensitive origin, unimportant sites. 🟩 Medium: Cross-origin request, no sensitive data. 🟦 High: Access tokens, real user limited data. 🟪 Critical: Cross-site access to admin account or critical data.
💡 Exploit Tips: Check `Access-Control-Allow-Origin` headers. Test with custom domains. Look for misconfigured wildcard `*` with credentials, or sensitive endpoints like `/api/user`, `/settings`.

Open Redirect / URL Redirect

⬜ Informative / N/A: Redirect endpoint, no real impact. 🟨 Low: Safe redirect, no user impact. 🟩 Medium: Phishing without real data. 🟦 High: Redirect login, capture token / session. 🟪 Critical: Social engineering, account compromise.
💡 Exploit Tips: Test parameters like `next=`, `url=`, `redirect=`. Use open redirect for phishing or CSRF chaining. Common endpoints: login redirects, payment confirmation pages.

Security Misconfiguration

⬜ Informative / N/A: Minor info exposure (banner, version). 🟨 Low: Exposures without real risk. 🟩 Medium: Access internal endpoints without privileges. 🟦 High: Modify sensitive settings / admin panel. 🟪 Critical: Full application / server compromise.
💡 Exploit Tips: Check default credentials, exposed admin pages, verbose errors. Review configuration files, headers, and backup endpoints.

Insecure Direct Upload / File Upload

⬜ Informative / N/A: Upload without impact, non-executable files test. 🟨 Low: Non-executable files, no impact. 🟩 Medium: Files visible to other users, no code execution. 🟦 High: Files can execute code or steal sessions. 🟪 Critical: RCE via uploaded files, full impact.
💡 Exploit Tips: Test file types, extensions, content-type headers. Upload scripts (PHP, JSP, ASPX) to reachable directories. Check file name normalization, temp directories, and image conversion endpoints.

Authentication / Session Issues

⬜ Informative / N/A: Error messages or redirects, no real impact. 🟨 Low: Error messages / incorrect redirect. 🟩 Medium: Logout / password reset on test account. 🟦 High: Access other users' accounts via token reuse. 🟪 Critical: Compromise real account / admin / API token.
💡 Exploit Tips: Test session fixation, token reuse, password reset endpoints. Inspect cookies, JWTs, and API tokens. Check for weak session IDs or missing expiry.

🚪 Improper Authorization

⬜ Informative: Public endpoint that should not exist, no sensitive data. 🟨 Low: Access non-critical resources (public profile, non-sensitive settings). 🟩 Medium: Modify your own data via simple bypass. 🟦 High: Access/modify other users' data. 🟪 Critical: Access admin functions or massive sensitive data.
💡 Exploit Tips: Check ID manipulation in URLs, API calls, and parameters. Test endpoints without proper token/role checks. Common targets: `/user/{id}`, `/admin/settings`, `/api/data/{id}`.

🔐 Privilege Escalation

⬜ Informative: Enumerate roles without direct access. 🟨 Low: Can see roles but cannot use them. 🟩 Medium: Access functions of non-admin users. 🟦 High: Access limited admin data or functions. 🟪 Critical: Gain full control as admin/root.
💡 Exploit Tips: Check role-related endpoints, modify user roles in requests, or attempt privilege upgrade via API. Look for misconfigured RBAC or missing role checks.

🌐 Subdomain Takeover

⬜ Informative: Exposed subdomain, but not controllable. 🟨 Low: Unused subdomain, minimal impact. 🟩 Medium: Can host own content but no sensitive data. 🟦 High: Can perform phishing or steal cookies from domain. 🟪 Critical: Full control, compromise users/applications.
💡 Exploit Tips: Check CNAME, NS records. Look for unclaimed services on GitHub, AWS, Azure, Heroku. Deploy malicious content or phishing pages on unclaimed subdomains.

💻 OS Command Injection

⬜ Informative: Suspicious parameter, no command execution. 🟨 Low: Can execute benign commands, no real impact (e.g., ping localhost). 🟩 Medium: Extract limited server info. 🟦 High: Access sensitive data or other processes. 🟪 Critical: Full RCE, complete server control.
💡 Exploit Tips: Check parameters reaching shell commands (`ping`, `convert`, `upload`). Try injection payloads (`;`, `&&`, backticks). Look for unsanitized inputs in system calls, cron scripts, and backend processes.

SSTI (Server-Side Template Injection)

⬜ Informative: Vector detected but filtered. 🟨 Low: Injection causes minor template crash only. 🟩 Medium: Can evaluate simple expressions (e.g., 2+2). 🟦 High: Access server variables, local files. 🟪 Critical: Code execution → full RCE.
💡 Exploit Tips: Test template fields (`{{ }}`, `{% %}`) and look for unsanitized input in user-facing templates. Common endpoints: form inputs, comment sections, profile pages.

Insecure Deserialization

⬜ Informative: Only vector found, no impact. 🟨 Low: Application crash with invalid payload. 🟩 Medium: Modify minor data (flags, roles). 🟦 High: Privilege escalation, access other users’ data. 🟪 Critical: Payload → code execution / RCE.
💡 Exploit Tips: Look for endpoints accepting serialized objects (JSON, XML, PHP, Java). Attempt tampering with object properties to escalate privileges or trigger code execution.

Race Condition

⬜ Informative: Only testing timing conditions. 🟨 Low: Duplicate requests → minor errors. 🟩 Medium: Create harmless duplicates (e.g., two messages). 🟦 High: Duplicate transactions/withdrawals limited. 🟪 Critical: Unlimited money transfer / access to infinite resources.
💡 Exploit Tips: Target endpoints handling transactions, counters, or resource allocation. Use rapid concurrent requests to trigger inconsistent states. Common targets: `/transfer`, `/checkout`, `/api/update`.

Business Logic Flaws

⬜ Informative: Strange functionality, no impact. 🟨 Low: Incorrect discount, minor loss. 🟩 Medium: Bypass non-critical restrictions. 🟦 High: Bypass payment / access unauthorized data. 🟪 Critical: Exploit → money transfer / full control.
💡 Exploit Tips: Analyze workflow and business rules. Test for bypasses in pricing, access control, or transaction logic. Common endpoints: `/checkout`, `/apply-discount`, `/redeem`.

XXE (XML External Entity)

⬜ Informative: Vector detected, no leak. 🟨 Low: Leak non-sensitive files (logs). 🟩 Medium: Access /etc/passwd or simple local files. 🟦 High: Exfiltrate API keys / DB config. 🟪 Critical: XXE → SSRF, RCE, internal system access.
💡 Exploit Tips: Test XML inputs with DOCTYPE and ENTITY declarations. Check endpoints accepting XML: SOAP APIs, import/export functions, and configuration uploads.
